| Feature / Concept             | C++ Version                              | Python Version                                                      | Notes / Differences                                                       |
| ----------------------------- | ---------------------------------------- | ------------------------------------------------------------------- | ------------------------------------------------------------------------- |
| **Class Declaration**         | `class Node { ... };`                    | `class Node:`                                                       | C++ uses `{}` and semicolons; Python uses indentation and `:`             |
| **Member Variables**          | `int data; Node* left; Node* right;`     | `self.data = val; self.left = None; self.right = None`              | Python uses `self.` to access instance variables                          |
| **Constructor**               | `Node(int val) { ... }`                  | `def __init__(self, val): ...`                                      | Python uses `__init__` instead of C++ constructor                         |
| **Pointers**                  | `Node*`                                  | Python references automatically                                     | Python does not have explicit pointers; `left` and `right` are references |
| **Memory Allocation**         | `new Node(nums[mid])`                    | `Node(nums[mid])`                                                   | Python automatically handles memory allocation (no `new`)                 |
| **Memory Deallocation**       | Manual (`delete`) if needed              | Automatic (garbage collection)                                      | Python frees memory automatically                                         |
| **Function Call / Recursion** | `buildBST(nums, low, mid-1)`             | `self.build_bst(nums, low, mid-1)`                                  | In Python, methods inside classes must use `self.`                        |
| **Base Case for Recursion**   | `if(low > high) return nullptr;`         | `if low > high: return None`                                        | `nullptr` vs `None`                                                       |
| **Inorder Traversal**         | `void Display(Node* root)`               | `def display(self, root):`                                          | Python uses indentation and `self.display` for recursion                  |
| **Printing**                  | `cout << root->data << " ";`             | `print(root.data, end=" ")`                                         | Python’s `print` handles output formatting; C++ uses `cout`               |
| **Array / Vector**            | `vector<int> nums = {...};`              | `nums = [...]`                                                      | Python lists vs C++ `vector`                                              |
| **Calling Methods**           | `object.Display(root);`                  | `solution.display(root)`                                            | Python uses dot notation; methods are called with lowercase (PEP8 style)  |
| **Return Type**               | Explicit pointers: `Node*`               | `Node` reference                                                    | Python handles references automatically, no explicit pointer type         |
| **Error Handling / Safety**   | Segfaults possible if pointer is invalid | Exceptions raised if object is None                                 | Python safer for null references                                          |
| **Recursion Depth Limit**     | Usually large (depends on stack)         | Default recursion limit ≈ 1000 (`sys.setrecursionlimit` can change) | Very deep trees may require increasing recursion limit in Python          |
